Git Cheat Sheet

Welcome to the Git Cheat Sheet! This resource is designed to help you quickly find and understand essential Git commands and concepts. Whether you're a beginner or an experienced developer, this cheat sheet will serve as a handy reference for your Git workflow.

Getting Started with Git

Git is a distributed version control system that allows developers to track changes in their codebase, collaborate with others, and manage different versions of their projects. Here are some fundamental concepts and commands to get you started:

  • Repository (Repo): A repository is a storage space where your project files and their version history are kept. You can create a new repository using the command: git init
  • Clone: To create a local copy of a remote repository, use the command: git clone [repository URL]
  • Status: Check the status of your working directory and staging area with: git status
  • Add: Stage changes for the next commit using: git add [file] or git add . to stage all changes
  • Commit: Save your staged changes to the repository with a descriptive message: git commit -m "Your commit message"
  • Push: Upload your local commits to a remote repository using: git push [remote] [branch]
  • Pull: Fetch and merge changes from a remote repository into your local branch with: git pull [remote] [branch]
  • Branch: Create a new branch to work on features or fixes without affecting the main codebase: git branch [branch name]
  • Checkout: Switch to a different branch or restore files: git checkout [branch name] or git checkout [file]
  • Merge: Combine changes from one branch into another: git merge [branch name]
  • Log: View the commit history with: git log
